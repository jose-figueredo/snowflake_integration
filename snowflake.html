Snowflake GitHub Integration

Complete guide for integrating GitHub repositories with Snowflake to automate SQL deployments, manage version-controlled database objects, execute remote scripts, and enable CI/CD workflows.

Overview Architecture Setup Objects SQL RBAC Automation Limitations

Overview

Snowflake GitHub Integration allows SQL scripts, Python files, and other project assets stored in GitHub repositories to be accessed directly inside Snowflake.

This enables:

  • Version-controlled database development
  • CI/CD for Snowflake projects
  • Automated deployment of SQL objects
  • Running SQL directly from GitHub
  • Using GitHub-hosted Python code in Snowpark stored procedures
  • Synchronizing Snowflake with GitHub branches and commits

Step 1 - GitHub Setup

  1. Create a GitHub repository (Public or Private).
  2. Create a Personal Access Token.
  3. Grant repository permissions.
  4. Copy and securely store the token.

For private repositories, the Personal Access Token is required for authentication.

Snowflake Objects

Object Purpose
Secret Stores GitHub credentials (Private repositories only)
API Integration Connects Snowflake with GitHub
Git Repository Represents the GitHub repository inside Snowflake
Repository Stage Internal stage used to browse repository contents

Private Repository Workflow

  1. Create Secret Object
  2. Create API Integration
  3. Create Git Repository Object
  4. Browse repository files
  5. Synchronize using FETCH
  6. Execute SQL directly from repository

Public Repository Workflow

Public repositories do not require a Secret object. Only an API Integration and Git Repository object are required.

Useful SQL Commands

Command Description
SHOW GIT REPOSITORIES List configured repositories
DESCRIBE GIT REPOSITORY Repository details
SHOW GIT BRANCHES View repository branches
LIST @repository/branch List repository files
ALTER GIT REPOSITORY FETCH Synchronize repository
DROP GIT REPOSITORY Delete repository object

Repository Features

  • Main branch support
  • Feature branch support
  • Tag browsing
  • Commit browsing
  • Repository synchronization
  • Download repository files
  • View repository metadata

RBAC Permissions

Permission Purpose
READ View repository contents
WRITE Run repository synchronization (FETCH)
USAGE (Database) Access database
USAGE (Schema) Access schema

Important:

  • WRITE requires READ permission.
  • Database and Schema USAGE privileges are also required.
  • WRITE does not allow modifying GitHub files.

Repository Stage vs Internal Stage

Automatically synchronized with GitHub Manual uploads

Executing SQL from GitHub

Snowflake can execute SQL files directly from the repository stage using: 

EXECUTE IMMEDIATE FROM
@repository/branches/main/sql/create_tables.sql;

Always synchronize the repository first using FETCH to retrieve the latest changes.

Snowpark Integration

GitHub-hosted Python files can be imported into Snowflake Stored Procedures.

Benefits:

  • Version-controlled Python code
  • Cleaner stored procedures
  • Reusable business logic
  • Snowpark support

Synchronization

Repository synchronization updates the Snowflake repository stage with the latest commits from GitHub.

Synchronization is performed using: 

ALTER GIT REPOSITORY repository_name FETCH;

This refreshes:

  • Branches
  • Tags
  • Commits
  • SQL Files
  • Python Files

Current Limitations

  • Repository origin cannot be changed after creation.
  • Repository is read-only inside Snowflake.
  • FETCH requires WRITE privilege.
  • Changes made in GitHub require manual synchronization.
  • Some UI features are still in Public Preview.
  • Object ownership is controlled through Snowflake RBAC.

Best Practices

  • Use AccountAdmin to create integration objects.
  • Store SQL scripts in version control.
  • Separate development using feature branches.
  • Synchronize repositories before deployments.
  • Grant least-privilege access.
  • Use GitHub with Snowpark for reusable Python code.
  • Implement CI/CD for automated database deployments.

Key Takeaways

  • GitHub integrates natively with Snowflake.
  • Supports both Public and Private repositories.
  • Uses Secret, API Integration, and Git Repository objects.
  • Enables version-controlled SQL deployments.
  • Supports branch, tag, and commit browsing.
  • Execute SQL files directly from GitHub.
  • Import Python modules into Snowpark stored procedures.
  • Repository synchronization keeps Snowflake up-to-date.
  • Designed for collaborative Data Engineering and CI/CD workflows.
